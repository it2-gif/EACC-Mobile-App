How To Host It
==============

You asked about Hostinger VPS versus normal shared hosting.

Short answer:

- Use Hostinger VPS if you want this app to run properly and professionally
- Do not rely on normal shared hosting for the backend


Why VPS Is The Correct Choice
=============================

Your project needs:

- A long-running NestJS backend
- Environment variables
- PostgreSQL access
- Firebase Admin credentials on the server
- API endpoints always available
- Process manager support such as PM2
- Reverse proxy such as Nginx
- SSL and domain control

That is what a VPS is for.


Why Normal Shared Hosting Is Not Recommended
============================================

Normal shared hosting is usually fine for:

- Simple PHP websites
- Static HTML pages
- Basic WordPress sites

It is usually not good for:

- NestJS backend hosting
- Long-running Node.js API processes
- Production-grade API control
- Real-time style app backends
- Easier debugging and deployment workflows

You might host only the Flutter web build on shared hosting if it is purely static,
but the backend should still be on a VPS or another backend host.


Best Practical Hosting Options
==============================

Option A: Hostinger VPS

Best if:

- You want one main server you control
- You want backend deployment freedom
- You want professional production hosting

Recommended use:

Best if:

My recommendation:

Use Hostinger VPS for backend and optionally for the Flutter web build too.
It gives you fewer platform limitations and a cleaner production setup.


How To Host On Hostinger VPS
============================

Step 1: Prepare the server

- Buy a Linux VPS
- Point your domain or subdomain to the VPS
- Install Node.js LTS
- Install Nginx
- Install PM2
- Install Git
- Install SSL with Let's Encrypt


Step 2: Upload the backend

- Clone the repository to the VPS
- Go to the backend folder
- Add production `.env` values
- Add the Firebase Admin credentials on the server only
- Run Prisma migration deploy
- Build the NestJS app
- Start it with PM2

Example backend flow:

- `npm install`
- `npx prisma migrate deploy`
- `npm run build`
- `pm2 start dist/src/main.js --name eacc-backend`


Step 3: Configure Nginx

- Proxy HTTPS requests from your domain to the NestJS backend port
- Example:
  - `https://api.yourdomain.com` -> `http://localhost:3000`


Step 4: Deploy the Flutter web app

- Run `flutter build web`
- Copy the `build/web` files to the web root on the VPS
- Serve them with Nginx
- Example:
  - `https://chat.yourdomain.com`


Step 5: Update production configuration

- Change Flutter backend base URL to the hosted API domain
- Confirm Firebase project settings allow the production web domain
- Deploy final Firestore and Storage rules


Step 6: Run production smoke tests

- Student login
- Teacher login
- Admin login after admin is finished
- Text chat
- Image upload
- Video upload
- Voice record and playback
- Notification delivery
- Logout and session restore
